# Content Generators Rebuild — Design Spec
**Date:** 2026-06-08  
**Status:** Approved

## Problem

All four content generators (G1–G4) are broken in three ways:
1. Hardcoded to 9 kitchen categories — 45 of 54 categories generate no content
2. `extractSpecs(name)` regex parses product names for specs → 3–5 fields, often empty → generators hallucinate when specs missing → 692 drafts deleted for quality
3. Old schema field references (`product._legacy.name`, `product.offers.affiliate_url`) — master-products.json uses flat fields

**Root cause of 692 deleted drafts:** spec coverage was 0% before enrichment. Now 1,901/1,921 products (99%) have 15–77 real Amazon spec fields in `product.specifications`. Generators must use this data.

## Scope

- 52 categories (54 total minus earbuds + bluetooth-speakers, both `zero_commission: true`)
- All 4 generators: G1 (category/brand hubs), G2 (reviews), G3 (comparisons), G4 (buying guides)
- New shared library: `lib/categories.js`
- Queue rebuild: `scripts/build-content-queue.js`
- Source: `data/master-products.json` (1,921 products, committed)

## Spec Rendering

### Old (broken)
```js
const specs = extractSpecs(product.name)  // regex on name string → 0–5 fields
```

### New
```js
function buildSpecTable(product, catSlug) {
  const raw = product.specifications || {}
  const keySpecs = CATEGORIES[catSlug]?.key_specs || []
  // Use key_specs order; fall back to first 12 fields from raw if no matches
  const keys = keySpecs.length
    ? keySpecs.filter(k => raw[k])
    : Object.keys(raw).slice(0, 12)
  if (!keys.length) return '<p style="color:#666;font-size:14px;">See Amazon for full specifications.</p>'
  return `<table ...>${keys.map(k => `<tr><td>${k}</td><td>${raw[k]}</td></tr>`).join('')}</table>`
}
```

Result: laptops get 8 real spec fields (processor, RAM, storage, screen, OS, battery, GPU). Air fryers get capacity, wattage, presets, etc. Zero hallucination — all values from Amazon.

## Generator Changes

### Common to all 4 generators

| Before | After |
|--------|-------|
| `product._legacy.name` | `product.name` |
| `product.offers.affiliate_url` | `product.affiliate_url` |
| `product.offers.external_id` / `product._legacy.asin` | `product.asin` |
| `product._legacy.review_count` | `product.review_count` |
| `product._legacy.price_segment` | `product.price_segment` |
| `require('../data/products/*.json')` per-cat | `require('../data/master-products.json')` once |
| `extractSpecs(name)` | `buildSpecTable(product, cat)` from lib |
| `KITCHEN_BEACHHEAD` 9-cat array | `ALL_CATS` 52-cat array from lib |
| Inline `CAT_LABELS`, `CAT_SIGNALS` objects | Imported from `lib/categories.js` |

Filter at load time: `products.filter(p => !p.zero_commission)`

## Queue Rebuild — `scripts/build-content-queue.js`

Replaces the current 9-cat queue (1,332 items) with 52-cat queue (~7,000 items).

### Content types per category

| Type | Count per cat | Logic |
|------|--------------|-------|
| `category_page` | 1 | Always |
| `brand_page` | top 5 brands with ≥2 products + trusted slug | ~3–5 per cat |
| `review` | all products | ~35 per cat |
| `comparison` | top 5 products per price_segment, group by segment | ~3–5 per cat |
| `buying_guide` | 3 per cat (budget/mid-range/premium) | always 3 |

### Priority scoring (unchanged formula)

```
priority_score = rankability * 100 + (commission_rate * 10) + (review_count / 100)
```

### Output shape per item

```json
{
  "rank": 1,
  "title": "Best Laptops in India 2026 — Prices, Reviews & Deals",
  "type": "category_page",
  "subtype": null,

  "commission_rate": 0.035,
  "revenue_estimate": { "low": 3200, "medium": 10500, "high": 31500 }
}
```

## Error Handling

- Products with `zero_commission: true` filtered at load, never reach generators
- Products with empty `specifications` (`{}`) → spec table shows "See Amazon for full specifications" — no hallucination
- Unknown category slug → `key_specs` fallback to first 12 spec fields
- WP API error → log + continue, don't abort batch
